Tailwind CSS: Inconsistent Behavior with Dynamic Color Classes

Issue Description

When using Tailwind CSS with dynamic color classes, certain hex color codes work perfectly while others fail to apply, despite using identical syntax and implementation.

Working Example

cardColor="[#008ffb]"  // Works perfectly for both text-[#008ffb] and bg-[#008ffb]
cardColor="[#ff9920]"  // Works perfectly
cardColor="[#d83b34]"  // Works perfectly

Non-Working Example

cardColor="[#FBCFE8]"  // Doesn't work
cardColor="[#E0F2FE]"  // Doesn't work
cardColor="[#1b2559]"  // Doesn't work
cardColor="[#111c44]"  // Doesn't work

Implementation

// Component usage
<CustomCard 
  title="Saved Venues"
  cardColor="[#008ffb]"
  addclassName=""
>

// Component implementation
<h2 className={`text-${cardColor} text-shadow-sm text-3xl font-mono`}>
  {title} 
</h2>
<div className={`h-[1px] -mt-1 mb-4 bg-${cardColor}`}> </div>

Observations

  • The syntax is identical for all color codes
  • Some colors work for both text and background
  • Some colors work partially (e.g., #d1f5f0 worked for background but not text)
  • Case sensitivity doesn't affect the outcome
  • The colors that work are already being used elsewhere in the project
  • The issue persists even when the non-working colors are used directly in other Tailwind classes in the same project

Questions

  1. Why do some hex colors work perfectly while others fail when using identical syntax?
  2. Is there a pattern to which colors work and which don't?
  3. Could this be related to how Tailwind processes dynamic class names with arbitrary values?
  4. Is there a difference in how Tailwind handles color codes in its internal color processing versus arbitrary values?

Environment

  • Next.js
  • Tailwind CSS
  • TypeScript
  • shadcn/ui components

Workaround

Currently using known working colors (e.g., #008ffb) as a reliable solution, but would like to understand the root cause of this behavior.
